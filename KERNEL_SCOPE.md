# Kernel Scope

## What This Kernel Is

The OPHI Diagnostic Kernel is a diagnostic and intervention framework for identifying, classifying, containing, and documenting failure in complex systems.

It is designed for environments where:
- execution pressure distorts constraints
- authority becomes ambiguous
- rollback may be weak or unavailable
- failure propagates across boundaries
- intervention must remain bounded and auditable

## What This Kernel Does

The kernel helps operators:

- define the boundary under review
- extract governing constraints
- identify failure signals
- classify recurring failure modes
- model propagation paths
- select bounded interventions
- preserve traceability and audit logic

## What This Kernel Is Not

This kernel is not:

- a generic project management framework
- a pure capability-expansion toolkit
- a substitute for domain expertise
- an autonomous decision-maker
- a compliance guarantee
- a replacement for formal legal, security, or safety review where required

## Output Standard

A valid kernel artifact should produce:

- a clear boundary definition
- a recognizable failure class
- a plausible propagation model
- a bounded intervention
- a record of uncertainty and rationale

## Exclusions

This kernel does not attempt to:
- automate every diagnosis
- reduce all organizational failure to a single taxonomy
- eliminate human judgment
- optimize systems without first preserving reversibility and constraint integrity

## Scope Discipline

If an artifact expands beyond diagnosis and bounded intervention into broad redesign, speculative strategy, or implementation planning, it should be split into:
1. diagnostic record
2. intervention record
3. downstream design work

The kernel exists to preserve clarity at the point of failure and intervention.

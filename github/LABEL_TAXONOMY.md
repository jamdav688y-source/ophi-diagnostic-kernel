# Label Taxonomy

This repository uses labels to separate doctrine work, failure analysis work, intervention work, documentation work, and release work.

## Core Labels
- doctrine
- failure-mode
- intervention-pattern
- case-study
- template
- documentation
- workflow
- schema
- governance

## Priority Labels
- priority:high
- priority:medium
- priority:low

## Status Labels
- status:blocked
- status:ready
- status:draft

## Release Labels
- release:v1.1
- release:v1.2

## Usage Rule

Use at least:
- one domain label
- one priority label
- one status label

for every serious issue.

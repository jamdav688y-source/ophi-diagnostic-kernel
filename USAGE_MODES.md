# Usage Modes

The OPHI Diagnostic Kernel can be used in several ways depending on the operator’s objective.

## 1. Audit Mode

Use the kernel to assess a system that already exists.

Best for:
- AI systems
- workflow review
- operational risk review
- governance diagnostics

Primary outputs:
- identified failure mode
- propagation path
- intervention recommendation
- audit trace

## 2. Intervention Design Mode

Use the kernel when a failure is already visible and a bounded change must be selected.

Best for:
- escalation design
- failure containment
- workflow correction
- rollback planning

Primary outputs:
- intervention pattern
- assumptions
- reversibility notes
- containment logic

## 3. Case Study Mode

Use the kernel to analyze a public system, public discourse, product failure, or organizational event.

Best for:
- thought leadership
- public examples
- teaching
- internal training

Primary outputs:
- structured case study
- failure classification
- bounded intervention analysis

## 4. Repo Bootstrap Mode

Use the kernel as a starting scaffold for another repository or governed system.

Best for:
- new governance repos
- diagnostic toolkits
- control-plane projects
- reusable templates

Primary outputs:
- inherited structure
- templates
- doctrine baseline
- audit-oriented folder organization

## 5. Training Mode

Use the kernel to teach diagnostic discipline.

Best for:
- operator onboarding
- governance education
- internal method training

Primary outputs:
- repeated case analysis
- consistent vocabulary
- reusable failure classes
- intervention literacy

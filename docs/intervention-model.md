# Intervention Model

## Objective

Translate diagnosis into bounded structural change.

## Intervention Principles

1. Interventions must address cause, not symptom.
2. Interventions must remain constraint-compliant.
3. Interventions should prefer reversibility under uncertainty.
4. Interventions must specify decision ownership.
5. Interventions that introduce new systemic risk require escalation.

## Intervention Classes

### Authority Rebinding
Used when ownership, decision rights, or stop authority are unclear.

### Escalation Insertion
Used when failures can spread without timely review or halt conditions.

### Constraint Hardening
Used when optimization pressure degrades non-negotiable boundaries.

### Rollback Path Design
Used when changes cannot be safely reversed within acceptable time.

### Observability Upgrade
Used when the system lacks enough visibility to detect divergence, misuse, or degradation.

## Required Fields for Any Intervention

- failure addressed
- structural mechanism changed
- cost introduced
- tradeoffs created
- new risks introduced
- reversibility profile
- escalation requirement
